## 테스트 실행 가이드

1. 로컬 개발 서버 실행: `npm run dev`
2. **게스트 흐름 우선 테스트** (FT-9 ~ FT-23): 로그아웃 상태에서 시작
3. High 우선순위 테스트부터 순서대로 진행
4. 클레임 흐름(FT-18 ~ FT-23)은 실제 Google 로그인 필요
5. DB 확인 필요한 테스트(FT-12, FT-13)는 Supabase Dashboard 또는 MCP 사용
6. 각 테스트 결과를 체크박스(⬜ → ✅/❌)에 기록
7. 실패 시 이슈 번호와 함께 기록

### 빠른 게스트 클레임 테스트 시나리오

```
1. 시크릿 모드 열기 (비로그인 보장)
2. 홈 → 면접 조건 선택 → 면접 시작 (FT-9)
3. 질문 1개 이상 답변 후 제출 (FT-11)
4. /complete 페이지에서 배너 확인 (FT-16)
5. "로그인하고 결과 보기" 클릭 (FT-18)
6. localStorage에 guestSessionId 저장 확인
7. Google 로그인 진행
8. /archive/:id 자동 이동 확인 (FT-20)
9. 클레임 완료 후 DB에서 user_id 업데이트 확인 (FT-12)
10. localStorage에서 guestSessionId 제거 확인 (FT-21)
```
